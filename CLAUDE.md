## КРИТИЧЕСКОЕ ПРАВИЛО — AI-комплаенс

Использовать ИИ-инструменты для написания кода можно и приветствуется. НО команда обязана
**полностью понимать** структуру, логику и архитектуру своего кода — своего или
сгенерированного ИИ. Если на Q&A или код-ревью человек не может объяснить логику —
баллы за техническую защищённость режутся **вплоть до 0**. Плагиат / код до 17 июля /
неспособность объяснить реализацию = дисквалификация.

→ Практическое следствие для Claude Code: **всегда объясняй нетривиальные решения** в
комментариях и коммит-сообщениях простым языком — команда должна суметь пересказать это
на защите своими словами.

## Продукт

- **Название:** EnergyPilot AI
- **Один пользователь:** семья с ограниченным бюджетом в городской квартире, Караганда
  (2–4 человека, 40–70 м², дифференцированный тариф, платит через Kaspi/Homebank/eGov).
- **Одна боль** (по правилу «один пользователь — одна острая боль»): счёт за
  электричество приходит раз в месяц постфактум, без объяснения структуры расхода и без
  возможности скорректировать поведение заранее.
- **Решение:** AI-ассистент, который на основе загруженных счетов / фото счётчика +
  короткой анкеты по квартире и технике даёт прогноз счёта, разбор структуры расхода,
  флаги аномалий и персональные рекомендации по экономии через чат.

## MVP-скоуп для хакатона (УРЕЗАННЫЙ — 4.5 дня, готовое важнее полного)

Делаем только это к дедлайну онлайн-этапа:

1. **Онбординг** — одна форма: квартира + техника + тариф (не растягивать на отдельные
   экраны регистрации и добавления квартиры).
2. **Загрузка счёта** — фото/PDF → OCR с обязательным fallback на ручную правку суммы /
   показаний. OCR никогда не должен блокировать флоу.
3. **Прогноз счёта** — ОДНА модель (Prophet или простая регрессия + сезонность), не
   тройной ансамбль LSTM/XGBoost/Prophet.
4. **Аномалии** — пороговое правило («рост X% без изменения привычек» → флаг), а не
   Isolation Forest / AutoEncoder — если не останется времени на ML-версию.
5. **Дашборд** — прогноз + разбивка по категориям + аномалии.
6. **AI-чат энергокоуч** — LLM со structured pipeline (профиль + прогноз + тариф + погода
   → рекомендация с оценкой экономии). НЕ голый chat-wrapper — это прямо оценивается
   критерием №2.

Явно ВНЕ скоупа для хакатона (упомянуть в питче как roadmap, не пытаться строить):
- Одновременный ансамбль из 3 ML-моделей.
- Isolation Forest / AutoEncoder для аномалий.
- Реальная интеграция с API ЭСО/eGov.
- Семейная геймификация, история сравнения месяцев (только на слайде монетизации).

## Правила работы в этом репозитории

- **Коммить часто и маленькими логичными кусками.** История коммитов — оцениваемое
  доказательство непрерывной работы, не делай один гигантский коммит перед дедлайном.
- Каждое нетривиальное решение — короткий комментарий или коммит-месседж с "почему",
  не только "что".
- README.md обязателен и должен включать: стек, архитектуру (текстовой схемы
  достаточно), инструкцию локального запуска, и **честную пометку**, что часть CustDev —
  синтетические гипотезы на основе открытых данных (KEGOC, Минэнерго, тарифные приказы),
  не реальные интервью. Регламент хакатона прямо разрешает синтетические данные при
  честном указании этого в README и на защите.
- Все пути отказа OCR/ML должны быть graceful — по этому же флоу будет записываться
  демо-видео, оно не должно падать посреди сценария.

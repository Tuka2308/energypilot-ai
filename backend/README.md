# EnergyPilot AI — backend

FastAPI-скелет под MVP-скоуп (см. корневой `CLAUDE.md`). Загрузка счёта
(`app/services/bills_service.py`) уже делает реальное распознавание;
прогноз/аномалии/чат пока возвращают моковые данные — подключаются
следующим шагом, без изменения контрактов роутеров.

## Как распознаётся счёт

Два пути по типу входа:

1. **Цифровой PDF** (типичная "Ведомость потребления электроэнергии" ЭСО) —
   это таблица с текстовым слоем, а не картинка. Читаем её структурно через
   `page.find_tables()` PyMuPDF и берём значения из конкретных колонок по
   заголовкам ("Сумма с НДС" → сумма к оплате, "Потреблено" → кВт·ч). OCR
   здесь НЕ применяется: он превращал бы точный текст в шум и путался в
   плотной таблице (из-за чего сервис однажды уверенно вернул год "2026"
   как сумму).
2. **Фото счётчика / сканированный PDF** без текстового слоя — здесь OCR
   (Tesseract) по ключевым словам рядом с числом.

Доверие проверяется не только диапазоном значений: в табличном пути сумма
принимается, только если непротиворечиво читается из своей колонки (итог с
НДС ≥ суммы без НДС); в OCR-пути нет жадного фолбэка "первое число в
диапазоне". Любая неоднозначность/низкая уверенность → `requires_manual_review:
true`, а не тихо-неверное число.

## Структура

```
app/
  main.py            # создание FastAPI-приложения, CORS, регистрация роутеров
  core/config.py      # настройки из env/.env
  routers/            # HTTP-слой: онбординг, счета, прогноз, аномалии, чат
  models/schemas.py   # Pydantic-контракты запросов/ответов
  services/           # бизнес-логика (bills_service.py — реальный OCR,
                       # остальное пока моки, один файл на фичу)
```

## Системные зависимости

OCR загрузки счёта использует `tesseract-ocr` через `pytesseract` — сам
бинарник pip не ставит, нужен отдельно:

```bash
# macOS
brew install tesseract
# для распознавания кириллицы (казахстанские счета на русском) —
# положить rus.traineddata в tessdata, brew ставит только eng/osd/snum:
curl -fsSL -o "$(brew --prefix tesseract)/share/tessdata/rus.traineddata" \
  https://github.com/tesseract-ocr/tessdata_fast/raw/main/rus.traineddata

# Debian/Ubuntu
sudo apt-get install tesseract-ocr tesseract-ocr-rus
```

В Docker-образе (`Dockerfile`) это уже прописано через `apt-get`.

**Честно про качество:** OCR-движок офлайн и бесплатный, но заметно слабее
облачных на неровных/тёмных фото — таков компромисс за отсутствие
ключей/оплаты на хакатоне. Табличный путь для цифровых PDF, наоборот,
точный (читаем текстовый слой). В обоих случаях при сомнении сервис не
подставляет случайные цифры, а возвращает `requires_manual_review: true`.

## Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload
```

Открыть `http://localhost:8000/docs` — Swagger со всеми эндпоинтами.

Через Docker — см. `docker-compose.yml` в корне репозитория.

## Тесты

```bash
pytest
```

реальной квитанции ЭСО (`tests/fixtures/sample_receipt_kz.pdf`): проверяет,
что сумма/потребление/период извлекаются верно и что мусорный вход уходит в
ручную проверку, а не в 500.
